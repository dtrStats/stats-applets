TITLE: Illustrating the binomial distribution
GOAL: Show the behavior of the binomial distribution as p and n change. Illustrate how through simulation one can increasingly more accurately recover the true probability mass function as the number of simulations increases.
MATH REQUIREMENTS (get these right -- double-check each one)
- Population generators:
Binomial: with n and p specified by the user.
For plotting the Binomial PMF use: P(X=k) = C(n,k) * p^k * (1-p)^(n-k)
Use log-space computation to avoid overflow:
log(P) = logC(n,k) + k*log(p) + (n-k)*log(1-p), then exp().
Compute logC using sum of logs, not factorial directly.
P(X<=k) = sum of P(X=j) for j=0..k.
- For each trial run: draw 1 value from the selected population. Repeat for the chosen number of trials.
- Display: Have a two panel stacked figure. Top panel shows a bar plot with the true probability mass function of the Binomial(n,p) distribution, bottom panel shows a bar plot of the simulated values sampled.
CONTROLS (each shows its current value on screen)
- Slider: number of trials n, range 5 to 200, step 1, default 5
- Slider: number of successes k, range 0 to n, step 1, default 0. Clamp k to max n.
- Slider: probability of success p, range 0.0 to 1.0, step 0.01, default 0.5
- Slider: number of simulations, range 100 to 5000, step 100,
default 100
- Buttons: Run Simulation, Reset
CANVAS PLOT REQUIREMENTS
True pmf: vertical bars for each k from 0 to n.
Highlight bars from 0 to k in a distinct color for P(X<=k).
X-axis label: "k". Y-axis label: "P(X = k)". Tick marks.
Simulated pmf: vertical bars for each k from 0 to n.
Highlight bars from 0 to k in a distinct color for P(X<=k).
X-axis label: "k". Y-axis label: "P(X = k)". Tick marks.
Keep the limits of the y-axis fixed 0 and max{P(X=k)}+0.1 across simulations
Choose judiciously some colors from the wessanderson::Zissou1 palette
NUMBERS PANEL (update after each Update Plot)
- True pmf: "P(X <= [k]) = [value]" (6 decimal places)
- Simulated pmf: "P(X <= [k]) = [value]" (6 decimal places)
- Simulated mean: "x bar = [value]" (4 decimal places)
- Simulated SD: "s = [value]" (4 decimal places)

EDGE CASES TO HANDLE

TITLE: Contrasting the Normal and Student's t Distributions
GOAL: Show the similarities and difference between the standard 
normal and the t distribution with n-1 degrees of freedom as the sample 
size grows for different levels of confidence 1-alpha.
MATH REQUIREMENTS (double-check every formula)
- Normal PDF: f(z) = (1/(sqrt(2*pi))) * exp(-0.5*(z^2)
- t PDF: f(t) = (Gamma(df+1)/(sqrt(pi*df)*Gamma(df/2))) * (1+t^2/df)^(-(df+1)/2),
Where Gamma(a,b) is the Gamma function evaluated at a and b, and df = n-1.
Compute both curves at 300+ points for a smooth curve.
Also for P(Z<=z) and P(T<=t): use numerical integration (Simpson’s rule or
trapezoidal rule with 500+ steps). Do NOT use approximations.
